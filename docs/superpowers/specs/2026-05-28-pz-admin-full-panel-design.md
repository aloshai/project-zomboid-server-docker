# PZ Admin UI — Full Management Panel

Date: 2026-05-28

Builds on [the initial admin UI](2026-05-28-pz-admin-ui-design.md). Upgrades the
minimal UI into a full management panel, staying inside the single-container /
supervisord / Dokploy-compose model.

## Features

### 1. Live console + command sending

The PZ dedicated server reads admin commands from **stdin**. `launch_pz.sh`
(generated by `entry.sh`) opens a FIFO read-write as the server's stdin so the
server never sees EOF and no separate holder process is needed:

```bash
PZ_STDIN=/tmp/pz-stdin
rm -f "$PZ_STDIN"; mkfifo "$PZ_STDIN"
exec ./start-server.sh ${ARGS} <> "$PZ_STDIN"
```

- `POST /console` writes one sanitized line (CR/LF stripped) to the FIFO.
- Console output already streams via the existing SSE log tail; the Dashboard
  gains a command input box.

### 2. Workshop mod management (`lib/workshop.js`)

- List installed Workshop items from
  `pz-dedicated/steamapps/workshop/content/108600/<id>` and parse each item's
  `mods/*/mod.info` to map Workshop ID → Mod ID(s) + names + maps.
- Add a Workshop ID; fetch title/preview via the public Steam
  `ISteamRemoteStorage/GetPublishedFileDetails` endpoint (no API key). Item
  downloads on next restart.
- Toggle which Mod IDs are active (writes `Mods=`); manage `WorkshopItems=`.
- Reorder both lists. Optional keyword search when `STEAM_API_KEY` is set.

### 3. Scheduler + backups (`lib/scheduler.js`, `lib/backups.js`)

- `node-cron` jobs in the webui. Config persisted to
  `Zomboid/.pz-admin/schedule.json`.
- **Scheduled restart:** cron + lead time → `servermsg` warnings, `save`, then
  `supervisorctl restart pzserver`.
- **Backups:** cron → `tar czf` of `Saves/` + `Server/` into
  `Zomboid/.pz-admin/backups/`, keep last N. Manual backup / download / restore
  (stop → extract → start) / delete.

### 4. Resource stats + file manager (`lib/stats.js`, `lib/files.js`)

- **Stats:** read cgroup (v2 `memory.current` / `cpu.stat`, v1 fallback) every
  ~5s, keep a rolling in-memory history; `GET /api/stats`; Dashboard renders a
  Chart.js line (CDN).
- **File manager:** all paths sandboxed under `ZOMBOID_DIR` with a
  resolve-and-prefix-check guard. List / view+edit text / download / upload
  (busboy) / delete / mkdir.

## Architecture notes

- New libs are small and single-purpose; routes live in `server.js` (or split
  into a `routes/` dir if it grows).
- New deps: `node-cron`, `busboy`. `tar` is the system binary; `fetch` is
  built-in (Node 20).
- All routes behind the existing auth. File manager and console are admin-only
  power tools — sanitize inputs, sandbox paths.
- Config/state persisted under `Zomboid/.pz-admin/` so it survives container
  recreation in the volume.

## Pages

- **Dashboard:** status, controls, **command box**, live log, **resource chart**.
- **Server / Sandbox:** unchanged (typed config forms).
- **Mods:** Workshop items + Mod IDs management, reorder.
- **Backups:** schedule config, backup list, manual actions.
- **Files:** sandboxed file browser/editor.

## Testing

- Unit tests for `workshop.js` (mod.info parsing) and the path-sandbox guard in
  `files.js`.
- Local smoke test of routes; container-specific bits (FIFO, cgroup, tar,
  supervisorctl) verified post-deploy via logs/UI.

## Out of scope (YAGNI)

- Multi-user/roles, in-panel Workshop keyword search without an API key,
  editing `spawnregions.lua` by hand (auto-handled).
